## 一、执行摘要（核心发现）

⚠️ **最重要的发现：提供的 `agent.db` 是一个"未初始化"的空库。**

- 文件为合法 SQLite 格式（头部 `SQLite format 3`），但仅 1 个数据库页（4096 字节），`sqlite_master` 中 **0 张表、0 个索引、0 行数据**。
- 真正的表结构定义在源码 `passive_agent/storage/db.py` 的 `SCHEMA` 常量中，由应用运行 `db.ensure_init()`（见 `main.py:35`、`orchestrator/loop.py:40`）时落地建表并写入种子合规规则。
- 由于该应用尚未针对此文件执行过 `ensure_init()`，库处于"出厂空态"。

**结论性判断**：当前 `agent.db` 不含任何被采集的被动侦察情报，也不可直接运行针对业务表的 SELECT（会报 `no such table`）。本报告的 Schema、规模、画像、指标均基于**源码中的权威 Schema（`db.py`）** 重建并验证，并对真实的"未初始化"状态做了如实标注。

> 补充说明：该项目真正的**原始情报数据**（资产清单、子域名、Hunter 资产、Wayback 历史、GitHub 泄露、DNS 等）位于 `FAFU/原始数据/`、`FAFU/*.md` 等文件中（CSV / XLSX / MD 格式），**并不在**这个 SQLite 库里。`agent.db` 本质上是工具的**控制/元数据平面**（合规护栏、审批流、审计轨迹、主体登记、采集结果指针、限流配额），而非原始情报仓库。即便库被填充，资产明细也以 `payload_json` 的 JSON 形式存储，而非展开成关系表。

## 六、数据质量备注

1. **【严重】数据库未初始化**：`agent.db` 为 0 表 0 行的空库，Schema 未落地。一切业务分析须待 `ensure_init()` 执行后才有数据。当前唯一可确认的内容是文件格式合法、未被占用。
2. **无外键约束**：全部关联（subject_id / task_id）仅靠命名约定，`foreign_key_list` 为空 → 存在孤儿行风险，引用完整性无法由数据库保证。建议在应用层或迁移脚本中补充逻辑校验。
3. **时间字段类型不一致**：8 张表用 `created_at TEXT`，而 `t_rate_quota.updated_at` 声明 `DATETIME`、`t_audit_log.ts` 为 `TEXT`。SQLite 实际均存为文本，但语义不统一，建议统一为 `TEXT(ISO-8601 UTC)`。
4. **软删除类型不一致**：`t_rate_quota.deleted` 为 `TINYINT`，其余为 `INTEGER`。功能等价（同为 INTEGER affinity），建议统一以减歧义。
5. **无二级索引**：高基数分析/过滤列（`subject_id`、`source`、`decision`、`action`、`status`、`risk_level`、`created_at`）均无索引。数据量增长后，本报告 §四 的 GROUP BY / 时间范围查询将全表扫描。建议针对常用分析维度补建索引。
6. **半结构化列未规范化**：`payload_json` / `state_json` 为 TEXT JSON，真实资产/子域名/敏感发现以 JSON 存储，标准 SQL 不可直接聚合，依赖 json1 扩展；且需确认运行环境 SQLite 是否编译 json1。
7. **唯一性约束有限**：仅 `task_id`、`ip` 有 UNIQUE；`result_id` 等无唯一约束，存在重复入库可能。
8. **软删除一致性**：各表均有 `deleted` 标志，查询须显式 `WHERE deleted=0`（本报告指标 SQL 已统一加上）。
