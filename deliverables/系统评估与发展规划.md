# 🏗️ 企业被动信息搜集 Agent — 系统评估与发展规划

> 评估日期: 2026-07-13 | 代码量: ~3,000 行 Python | 测试: 11 个测试文件

---

## 一、系统现状评估

### ✅ 强项

| 维度 | 评分 | 说明 |
|------|:----:|------|
| **架构设计** | ⭐⭐⭐⭐⭐ | 8个R模块清晰分层，R1合规→R2核验→R3枚举→R4审批→R5台账→R6网关→R7采集→R8报告 |
| **合规安全** | ⭐⭐⭐⭐⭐ | fail-closed 设计，所有出站必经 R1 拦截，OpenSCA 台账零漏洞依赖 |
| **可扩展性** | ⭐⭐⭐⭐⭐ | 适配器模式（adapter.py）、策略模式（rules.py）、管道模式（pipeline.py） |
| **测试覆盖** | ⭐⭐⭐⭐ | 11 个测试文件覆盖各模块，含集成测试和压力测试 |
| **审批流程** | ⭐⭐⭐⭐ | 三级审批+断点快照，支持续跑 |
| **代理网关** | ⭐⭐⭐⭐ | IP池+频控+分片提交，已具备生产级网关雏形 |

### ⚠️ 待优化

| 维度 | 问题 | 优先级 |
|------|------|:------:|
| **数据源采集** | `adapter.py` 使用 mock 数据，没有接入真实被动源 | **P0** |
| **编排流程** | `orchestrator/loop.py` 是单轮线性采集，缺乏多轮递进融合 | **P0** |
| **去重融合** | 没有跨源去重引擎，多源数据会重复 | **P1** |
| **指纹识别** | 无 CMS/邮件/一卡通/认证系统指纹库 | **P1** |
| **Web 面板** | `static/index.html` 存在但功能未完整对接后端 API | P2 |
| **文档** | 缺乏 API 文档和部署指南 | P2 |
| **CI/CD** | 没有持续集成配置 | P3 |
| **容器化** | 没有 Dockerfile | P3 |

## 二、模块级深度评估

### R1 — 合规引擎 ✅ 完善

```
位置: compliance/engine.py + compliance/rules.py + common/compliance_client.py
逻辑: fail-closed，黑白名单双检
问题: 规则目前硬编码在 rules.py 中，没有规则热加载
建议: 改为 YAML 配置规则，支持运行时重载
```

### R2 — 核验流水线 ✅ 架构完善，实现需补

```
位置: verifier/pipeline.py + verifier/layers.py
现状: 4层核验架构（业务匹配/DNS存活/时效性/多源佐证）
问题: DNS存活层(layers.py)需要接入真实DNS解析
建议: 接入 dnspython 实现真实校验
```

### R3 — 主体枚举 ⚠️ 最需增强

```
位置: enumerator/engine.py + enumerator/adapter.py
现状: adapter._mock_relations() 返回 mock 数据
问题: ❌ 无法真实发现企业的子公司/关联公司
建议: 接入 ENScan_GO 或企查查/爱企查 API
```

### R4 — 审批服务 ✅ 完善

```
位置: approval/service.py + approval/model.py + approval/snapshot.py
现状: 三级审批+断点快照+续跑，功能完整
建议: 增加审批超时自动降级机制
```

### R5 — 台账登记 ✅ 完善

```
位置: inventory/registry.py + inventory/model.py
现状: 预置12个开源/自研组件台账，支持一键出具自研占比证明
建议: 增加 SPDX/SWID 标准导出
```

### R6 — 代理网关 ✅ 完善

```
位置: gateway/proxy.py + gateway/ip_pool.py + gateway/ratelimiter.py
现状: IP池轮询+滑动窗口频控+分片提交
问题: 缺少代理健康检查和自动切换
建议: 加入代理连通性心跳检测
```

## 五、技术债务

| 项 | 位置 | 说明 | 影响 |
|----|------|------|:----:|
| mock 数据 | `enumerator/adapter.py` | `_mock_relations()` 返回假数据 | 当前系统无法实际运行 |
| 硬编码规则 | `compliance/rules.py` | 黑白名单在代码中硬编码 | 修改需重启 |
| 单轮循环 | `orchestrator/loop.py` | 只有一个 for 循环处理主体 | 无法多轮递进 |
| 前端未对接 | `static/index.html` | 静态页面存在但没有实际 API 调用 | 面板不可用 |
| 无部署文档 | — | 缺少 README 部署说明 | 新人上手困难 |

## 七、一句话总结

> **系统架构是专业的，合规设计是严格的，但核心采集层还在用 mock 数据。把比赛淬炼的 15 种真实数据源接入、把 8 轮方法论编码为 Playbook、把指纹识别做成签名库——这就是从"框架"到"引擎"的关键一跃。比赛第一名验证了方法论可行，接下来就是工程化落地的问题。**
