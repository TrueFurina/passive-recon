# 设计令牌文档（Design Tokens Spec）
**项目**：企业被动信息搜集 Agent · 系统架构图解页
**视觉调性**：深蓝科技安全风（Tech Utility 学派定制）
**交付形态**：单页纵向长图（桌面优先 1920×1080，可导出 PNG/SVG）
**版本**：v1.0 · 已对齐「合规红线 / 三级审批 / 算力 60·30·10」三大主角元素

---

## 一、候选方案推荐

| 方案 | 设计系统 | 匹配度 | 契合点 | 取舍 |
|------|---------|--------|--------|------|
| A（选定） | **Supabase** | ★★★★★ | 深蓝近黑底 + 标志青绿 `#3ECF8E` 高亮；产品 DNA 即「数据库 / 关系图谱 / 开源工具链」，与本项目的 Neo4j 关联图谱、开源工具留档天然同构；克制发光、低对比网格的纪律可直接复用为 Anti-Slop 范式 | 其绿偏翠，需将高亮微调为更冷的青绿 `#2DD4BF` 以契合「攻防/科技」而非「SaaS 友好」 |
| B | **HashiCorp** | ★★★★☆ | 基础设施编排视觉语言，「分层架构 / 防御体系秩序感」表达成熟；企业级安全可信气质贴合政企受众 | 主色偏紫红，需大幅改色；拓扑/图谱语义弱于 Supabase |
| C | **Linear** | ★★★☆☆ | 极简精密、降认知负荷大师，冷静克制；适合评委「重严谨」诉求 | 近纯黑单色、缺青绿高亮，攻防科技感不足，仅作排版/密度纪律参考 |

**最终选定：方案 A — Supabase 视觉谱系（深蓝 + 青绿）做基底，融合 HashiCorp 的「分层秩序」与 Linear 的「降认知负荷间距」。**
定制理由：① Supabase 的深蓝 + 青绿是本需求「深蓝科技安全风」原生最接近的设计基因；② 其图谱/开源产品语义与本项目 Neo4j + 开源留档逐一呼应，避免视觉与内容割裂；③ 其「发光克制 + 网格低对比」的纪律恰好满足赛事视觉对「可信、不浮夸」的要求。

### 2.8 层级区分策略（四层微差明度 + 左色轨）
四层同属深蓝家族，用 **「明度阶梯 + 左侧 3px 色轨」** 双重区分，避免依赖彩虹色：
- 顶层：`--bg-layer-top` + 左轨 `#38BDF8`（电光蓝，最高调度层）
- 中层：`--bg-layer-mid` + 左轨 `#2DD4BF`（青绿，规划中枢）
- 下层：`--bg-layer-lower` + 左轨 `#7DD3FC`（浅蓝，采集执行）
- 底层：`--bg-layer-base` + 左轨 `#475569`（灰蓝，存储地基）
- 明度走向：顶层最亮 → 底层最沉，形成「由上至下收敛入地」的视觉重力，契合分层架构的「地基感」。

### 3.3 字距 / 行高规范
- Display：字间距 `-0.02em`；区块小标签（如「LAYER 01」）用大写 `+0.08em`。
- 等宽标签：字间距 `+0.01em`，保持技术质感。
- 正文行高 1.6，卡片内文 1.5，避免大屏下过松。

### 3.4 中英混排等宽约定
- 所有开源工具名、参数、API、端口、代码一律走 `--font-mono` 并包裹 `<code>` 小片（`--bg-chip` 底 + `--text-secondary` 字）。
- 等宽 token 紧邻中文时，左加 `0.25em` 间距（`margin-left:.25em`）保证呼吸感。
- 工具名清单行用 `11.5px` 等宽、行高 1.6，纵向排列，呈现「留档清单」质感。

## 四、间距与布局令牌（Spacing & Layout Tokens）

### 4.1 间距标尺
```
--space-xs:4px  --space-sm:8px  --space-md:12px  --space-base:16px
--space-lg:24px --space-xl:32px --space-2xl:48px --space-3xl:64px
```

### 4.2 容器与骨架
- `--container-max: 1200px`：内容居中（大屏下保持可读行宽；网格背景全幅铺底）。
- 层区块内边距：`--space-xl --space-lg`（上下 32 / 左右 24），桌面下可放宽至 36/28。
- 卡片间距：`--space-md`（12px）；卡片内边距 `--space-base`（16px）。
- 区块间纵向留白：`--space-3xl`（64px），形成 5 区块清晰分段。
- 层间留白：`--space-lg`（24px），层间以 **中央竖向主轴（电光蓝 `--grid-line-strong`）** 串联，节点用圆点标记。

### 4.3 纵向分层骨架分隔规范
- 层与层之间：24px 留白 + 1px 分隔线 `rgba(148,163,184,.12)`（仅在大区块边界使用，层内不用）。
- 层级缩进：下层四集群卡相对层标题缩进 `--space-lg`（24px），体现「子任务下发」从属关系。
- 网格背景：全幅 `background-image` 双线性渐变，步长 32px，色值 `--grid-line`（极低对比，仅暗示坐标系）。

## 五、组件规范（Component Specs）

### 5.1 层级区块容器（Layer Block）
- 底：`var(--bg-layer-X)`；左 3px 色轨（见 2.8）；圆角 `14px`；内边距 `32px 28px`。
- 标题栏：层标题（20px/600）+ 左侧「LAYER 0X」等宽小标签（`--font-mono`，`--text-muted`）+ 右侧「100% 自研 / 开源封装」状态徽标。
- 默认无投影，仅左轨与微提亮区分；悬停态加 `--glow-accent`（克制）。

### 5.2 卡片（Cluster Card / Agent Card）
- 底：`var(--bg-card)`；描边 `1px solid rgba(148,163,184,.14)`；圆角 `10px`；内边距 `16px`。
- 四集群卡（下层）：等宽工具名清单（11.5px）纵向排，底加 `--bg-chip` 片。
- Agent 卡（中层 LLM 四步）：步骤序号用青绿圆点，文字 13px。
- 核验四步卡（底层）：步骤编号 `①②③④` 青绿，含「≥2 源」强调标红。

### 5.3 审批三态标识（Badge / Pill）
- 形状：圆角 `999px`，内边距 `4px 10px`，等宽 `11.5px`。
- 低（自动）：字 `#2DD4BF` / 底 `rgba(45,212,191,.12)` / 边 `rgba(45,212,191,.40)`，图标 ✓。
- 中（半自动）：字 `#38BDF8` / 底 `rgba(56,189,248,.12)` / 边同蓝，图标 ⟳。
- 高（人工）：字 `#FF4D4F` / 底 `rgba(255,77,79,.12)` / 边同红，图标 ⚠（警示红为合规边界，允许稍强描边）。

### 5.4 算力比例条 / 环形图（60·30·10）
- 水平比例条：三段拼接，A `#2DD4BF`(60%) → B `#38BDF8`(30%) → C `#64748B`(10%)，段间 2px 间隙。
- 刻度标注：每段上方等宽微注标占比（「A 60%」），主色字。
- 环形图（可选）：同三色占比环，中心标注「25min 回收 · 5min 同步」。

### 5.5 合规红线横幅（Compliance Banner）
- 位置：区块①顶部，通栏。
- 底：`var(--bg-banner)`（暗红）；左 4px 实线 `--color-danger`；圆角 `10px`。
- 文案：主文 `--text-on-banner`（浅红），关键词（如「被动」「不主动发包」）用 `--color-danger-light` 加粗。
- 右侧放静态「合规校验通过」标（青绿 ✓），形成红→绿的安全暗示，但**不**用动画。

### 5.6 拓扑小图（Neo4j 节点-连线）
- 节点：圆角矩形 / 圆，底 `--bg-card-2`，描边 `--color-accent`(.4)，字 11.5px 等宽。
- 连线：SVG `stroke` 用 `--color-primary`，`opacity:.55`，`stroke-width:1.5`；重点关联线加 `--glow-primary`（克制）。
- 布局：力导向留白，避免交叉；资产关联用青绿，跨源验证用蓝。

### 5.7 流程箭头（核验四步串联）
- 样式：1.5px 电光蓝实线 + 三角箭头，节点为青绿圆点（编号）。
- 纵向或横向串联，配等宽步骤标签；禁止用「雷达/扫描」类动画箭头。

### 5.8 锚点导航（顶部四个锚点）
- 顶部 sticky 条，`--bg-page` 80% 毛玻璃，4 个锚点 pill。
- 默认：字 `--text-secondary`；当前锚点：字 `#E6EDF3` + 底部 2px 青绿下划线 + 轻微 `--glow-accent`。
- 锚点文案：①架构总览 ②四层主图 ③采集范围 ④开源留档 / ⑤图例算力（按 5 区块对应）。

## 六、Anti-Slop 自检

| 检查项 | 本令牌做法 |
|--------|-----------|
| 发光不廉价 | 仅 `--glow-accent/--glow-primary` 两处、低透明度（.30/.35）、仅焦点/边界用，绝不全页泛光 |
| 无彩虹渐变 | 全站仅「深蓝家族 + 青绿 + 电光蓝 + 单一警示红」四色系，层级区分用明度而非色相跳跃 |
| 网格低对比 | `--grid-line` 仅 `.06` 透明度，仅暗示坐标系，不抢主体 |
| 等宽强化技术质感 | 工具名/参数/代码全量等宽 + chip 底，呈现「留档/终端」真实感，规避「塑料 SaaS」感 |
| 间距降认知负荷 | 区块间 64px、层间 24px、卡片 12px，信息密集但有呼吸（Linear 纪律） |
| 真实数据密度 | 工具名逐一准确（Subfinder/OneForAll/crt.sh/FOFA-Hunter/七麦…），无占位假数据 |
| 合规可信暗示 | 红→绿安全色序、静态标而非动画，规避「主动扫描」误读 |
| 禁区守住 | 无任何雷达扫描 / 端口矩阵 / 旋转动画；开源名零拼写错误 |

## 原型构建师 · 三条铁律
1. **色彩只走四系**：深蓝底 + 青绿(高亮/主动) + 电光蓝(连线/半自动) + 警示红(合规/人工)，绝不加第五个色相，发光透明度永不超过 .35。
2. **开源工具名零容错**：所有采集工具名照 DESIGN 文档逐字等宽呈现，拼写/大小写一旦错即破坏「可信」主线。
3. **禁主动扫描语义**：任何雷达扫描、端口矩阵、旋转/脉冲动画一律禁用；合规红线用静态红→绿标表达「被动·合规」，不用动态暗示。
