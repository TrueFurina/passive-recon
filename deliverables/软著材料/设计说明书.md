## 一、系统概述

### 1.1 系统目标

企业被动信息搜集 Agent 是一款基于纯被动情报源的网络安全资产发现与管理系统。
其核心目标为：在不主动接触目标系统的前提下，通过多源被动情报数据聚合，
实现对企业数字资产的全面发现、风险识别与合规管控。

### 1.2 设计原则

| 原则 | 说明 |
|:-----|:------|
| 纯被动 | 零主动连接目标系统，不产生目标日志 |
| 一通百通 | 输入任意中文企业/高校名，自动推断域名并全源采集 |
| fail-closed | 合规拦截引擎默认阻断，白名单放行 |
| 可插拔 | 数据源适配器可动态注册，支持热切换 |

## 三、核心模块设计

### 3.1 域推断引擎（domain_db.py）

**功能**：输入中文企业/高校名称，自动推断其互联网主域名。

**覆盖范围**：
- 高校精确映射：350+ 所中国高校（含全部985/211/双一流）
- 企业精确映射：100+ 中国头部企业
- 算法推算：拼音首字母缩写 + 常用 TLD 后缀

**推断优先级**：
1. 已经是域名格式 → 直接返回
2. 企业精确匹配 → .com / .com.cn
3. 高校精确匹配 → .edu.cn
4. 高校模糊匹配（含"大学"/"学院"关键词）
5. 算法推算 → 拼音缩写 + 合理 TLD

### 3.2 采集器模块（collector/sources.py）

**被动数据源列表**：

| 采集器 | 源类型 | 采集内容 | 认证方式 |
|:-------|:-------|:---------|:---------|
| CrtshCollector | 证书透明日志 | 子域名 | 免费 |
| HackerTargetCollector | DNS 查询 | 子域名+IP | 免费 |
| OTXCollector | 威胁情报 | 被动 DNS | 免费/API Key |
| URLScanCollector | 网页快照 | 子域名+IP | 免费 |
| HunterCollector | 空间测绘 | 子域名+IP+端口+技术栈 | 多 Key 轮询 |
| FofaCollector | 空间测绘 | 子域名+IP | email+Key |
| QichachaCollector | 企业工商 | 企业信息+信用代码+法人 | AppKey+Secret |
| SecurityTrailsCollector | DNS 历史 | 子域名 | API Key |
| ReverseDnsCollector | IP 反向 | 域名 | 免费 |
| IcpCollector | ICP 备案 | 备案信息 | 免费 |

### 3.3 采集管理器（collector/manager.py）

**核心流程**：

```
collect(name, domain, sources)
  ├─ 自动推断域名 (如未指定)
  ├─ 构建采集器列表
  ├─ 并发采集 (ThreadPoolExecutor 并行多个数据源)
  ├─ 去重 (value + asset_type)
  ├─ IP 补全 (并发 DNS 解析)
  ├─ 风险检测 (端口/技术栈/子域名分析)
  └─ 返回 CollectReport
```

### 3.4 合规引擎（compliance/engine.py）

**六条红线（R1-R6）**：

| 红线 | 拦截规则 | 响应码 |
|:-----|:---------|:-------|
| R1 | 主动扫描/HTTP/TCP 发包 | 010001 |
| R2 | DNS 仅解析，禁止 socket 连接 | — |
| R3 | 出站前必经合规校验 | — |
| R4 | 高价值关键词人工复核 | — |
| R5 | 开源台账定期导出 | — |
| R6 | 频控硬闸≤95% | 010002 |

**实现方式**：所有采集器出站前调用 `compliance_client.check(PASSIVE_QUERY)`，
未放行则 throw `PermissionError`（fail-closed）。

### 3.5 风险检测引擎（manager.py _detect_risks）

自动化风险发现包含三个维度：

1. **高危端口检测**：21(FTP)、23(Telnet)、3389(RDP)、6379(Redis) 等
2. **系统暴露面检测**：WebVPN、OA、mail、ecard、CAS、Jenkins 等
3. **技术栈风险**：Synjones 一卡通、Coremail 邮件、DES 加密等

## 四、数据模型

### 4.1 核心数据实体

```
AssetRecord（资产记录）
├─ value: str          # 资产值（域名/IP/邮箱）
├─ asset_type: enum    # subdomain/ip/organization/...
├─ source: enum        # 数据源（crt.sh/hunter/...）
├─ ip: str             # IP 地址
├─ port: int           # 端口
├─ tech_stack: list    # 技术栈
├─ title: str          # HTTP 标题
└─ tags: list          # 标签

CollectReport（采集报告）
├─ enterprise: str     # 目标企业名
├─ domain: str         # 主域名
├─ total_records: int  # 总资产数
├─ records: list       # AssetRecord 列表
├─ sources_used: list  # 数据源列表
├─ errors: list        # 错误/风险
└─ to_table()          # 报告输出
```

### 4.2 数据库表结构

| 表名 | 用途 | 关键字段 |
|:-----|:------|:---------|
| t_collect_asset | 采集资产记录 | enterprise, asset_value, asset_type, source_name, ip |
| t_audit_log | 合规审计日志 | action, decision, reason_code, trace_id |
| t_compliance_rule | 合规规则 | action_type, decision, enabled |
| t_approval_task | 审批任务 | task_id, biz_type, risk_level, status |
| t_subject | 主体枚举 | enterprise, name, relation, credit_code |

### 5.3 数据源适配器接口

```python
class SourceAdapter(ABC):
    def collect(self, query: CollectQuery, trace_id: str) -> CollectResult: ...
    def _check_compliance(self, trace_id: str) -> None: ...
    def is_available(self) -> bool: ...
```

所有适配器必须：
1. 实现 `collect()` 抽象方法
2. 出站前调用 `_check_compliance()`（R1 关隘）
3. 返回标准 `CollectResult`

## 六、数据流

### 6.1 采集主流程

```
用户输入 "北京大学"
    │
    ▼
域推断引擎 → pku.edu.cn
    │
    ▼
采集管理器并发调用 10 个数据源
    │
    ├─ crt.sh ────┬─ 50+ 子域名
    ├─ Hunter ────┼─ 200+ 资产（含IP/端口/技术栈）
    ├─ 企查查 ────┼─ 企业信用代码/法人/联系电话
    ├─ URLScan ───┼─ 30+ 子域名
    └─ …… ───────┘
    │
    ▼
去重 → IP补全 → 风险检测
    │
    ▼
输出报告（终端/Excel/数据库）
```

### 6.2 合规拦截流程

```
采集器出站请求
    │
    ▼
compliance_client.check(PASSIVE_QUERY)
    │
    ├─ ALLOW → 正常调用外部 API
    │
    └─ BLOCK → PermissionError (fail-closed)
                    │
                    ▼
              写 t_audit_log 审计日志
```

## 七、安全设计

### 7.1 纯被动红线

- 禁止任何形式的主动扫描（端口扫描、HTTP 探测、TCP 连接）
- DNS 仅使用 `resolver.resolve()` 被动解析
- 所有出站调用必须经 R1 合规校验
- 违规行为即时代码拦截 + 审计日志留痕

### 7.2 密钥管理

- API Key 通过 `config.json` 统一配置
- 支持多 Key 轮询（Hunter 5 Key）
- 支持多环境配置（pydantic-settings）
