## 一、软件概述

### 1.1 产品定位

企业被动信息搜集 Agent（Enterprise Passive Reconnaissance Agent）是一款面向网络安全领域的**纯被动信息资产搜集与管理平台**。通过集成多源被动情报数据，在不主动接触目标系统的前提下，完成企业资产发现、风险识别、合规管控等任务。

### 1.2 核心功能

| 功能模块 | 说明 |
|:---------|:------|
| 资产采集 | 10 个被动数据源并行采集（crt.sh、Hunter、FOFA、企查查等） |
| 域名推断 | 输入中文企业名自动推断域名（覆盖 350+ 高校、100+ 企业） |
| 风险检测 | 基于端口、技术栈、子域名的自动化风险发现 |
| 批量处理 | 文件批量目标一键全源采集 |
| Excel 导出 | 多 Sheet 资产报告导出 |
| Web 管理面板 | FastAPI 可视化界面 |
| 合规审计 | R1-R6 六级红线合规体系，fail-closed 拦截 |

### 1.3 应用场景

- 护网/攻防演练前的外网资产盘点
- 高校/企业数字化资产测绘
- 安全审计中的资产梳理
- 红队评估的被动信息收集

## 三、快速入门

### 3.1 单目标全量采集

```bash
python cli.py collect 北京大学
```

**执行效果：**
```
🎯 目标: 北京大学
🌐 自动推断域名: pku.edu.cn
📡 开始采集: 北京大学 (pku.edu.cn)，启用 8 个数据源
  [crt.sh] → 50+ 子域名
  [hunter] → 200+ 资产
  [urlscan] → 30+ 子域名
  [qichacha] → 企业工商信息
📊 总资产: 280+ 条
💾 已落库到数据库
```

### 3.2 采集并导出 Excel 报告

```bash
python cli.py collect 北京大学 --sources "crt.sh,hunter" --export 北大资产报告.xlsx
```

生成 Excel 包含三个 Sheet：
- **资产清单**：所有发现的域名、IP、端口、技术栈
- **统计总览**：资产类型分布、IP 数、数据源
- **风险发现**：自动检测的安全风险

### 3.3 批量采集

```bash
# 准备目标文件（每行一个）
echo 北京大学 > targets.txt
echo 清华大学 >> targets.txt
echo 浙江大学 >> targets.txt

# 批量执行
python cli.py batch targets.txt --sources "crt.sh,hunter" --export 批量结果.xlsx
```

## 四、命令行参考

### 4.1 命令总览

| 命令 | 功能 | 使用示例 |
|:-----|:------|:---------|
| `collect` | 一通百通全源采集 | `python cli.py collect 北京大学 --export 报告.xlsx` |
| `batch` | 批量采集 | `python cli.py batch targets.txt` |
| `diff` | 资产变化追踪 | `python cli.py diff 北京大学 ./旧 ./新` |
| `domain-info` | 域名推断 | `python cli.py domain-info 北京大学` |
| `list-sources` | 数据源列表 | `python cli.py list-sources` |
| `icp` | ICP 备案查询 | `python cli.py icp pku.edu.cn` |
| `qichacha-detail` | 企查查工商详情 | `python cli.py qichacha-detail 12350000726430923A` |
| `qichacha-verify2` | 二要素核验 | `python cli.py qichacha-verify2 信用代码 企业名` |
| `qichacha-verify3` | 三要素核验 | `python cli.py qichacha-verify3 信用代码 企业名 法人` |
| `import-path` | 历史数据导入 | `python cli.py import-path ./FAFU` |
| `enumerate` | R3 主体枚举 | `python cli.py enumerate 北京大学` |
| `audit-queue` | 合规审计 | `python cli.py audit-queue` |

### 4.2 collect 命令参数

```
用法: python cli.py collect [--domain 域名] [--sources 源列表] [--export 路径] <目标名>

参数:
  目标名             必需，企业/高校中文名或域名
  --domain          可选，手动指定域名（不传则自动推断）
  --sources         可选，数据源逗号分隔（默认全部）
  --export          可选，导出 Excel 报告路径

示例:
  python cli.py collect 北京大学
  python cli.py collect 阿里巴巴 --domain alibaba.com
  python cli.py collect 北京大学 --sources "crt.sh,hunter" --export 报告.xlsx
```

### 4.3 batch 命令参数

```
用法: python cli.py batch [--domain 域名] [--sources 源列表] [--export 路径] <文件>

参数:
  文件              必需，目标列表文件（每行一个）
  --domain          可选，统一域名
  --sources         可选，数据源逗号分隔
  --export          可选，导出汇总 Excel
```

## 七、常见问题

### Q1: 为什么采集结果为空？

可能原因：
1. 网络不通——检查能否访问外网
2. API Key 过期——检查 config.json 中密钥
3. 目标域名不存在——用 `domain-info` 验证

### Q2: 如何验证 R1 合规拦截？

```bash
python cli.py compliance-status
# 主动动作 ACTIVE_SCAN -> allowed=False BLOCK ✅
```

### Q3: 数据存在哪里？

SQLite 数据库存储在 `data/agent.db`，可用任何 SQLite 工具查看。

### Q4: 如何导入已有资产数据？

```bash
python cli.py import-path ./已有数据目录
```

支持 Markdown 表格、JSON、CSV 格式。
