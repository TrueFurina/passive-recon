## 三、核心流程：8轮 Playbook 编排

```python
# playbooks/rounds.py

ROUND_8 = [
    {
        "round": 1,
        "name": "基础信息收集",
        "collectors": ["crtsh", "urlscan", "dns_basic"],
        "sources": 3,
        "output": "foundation_domains"
    },
    {
        "round": 2,
        "name": "深度扩展",
        "collectors": ["fofa_shodan", "ip138", "web_search"],
        "sources": 3,
        "fusion": True,          # 与第1轮融合去重
        "output": "expanded_domains"
    },
    {
        "round": 3,
        "name": "多源API补充",
        "collectors": ["hackertarget", "otx", "github"],

    },
    {
        "round": 4,
        "name": "技术指纹识别",
        "collectors": ["http_probe"],
        "fingerprint": ["cms", "mail", "card", "auth"],
        "output": "tech_stack"

    {
        "round": 5,
        "name": "社交资产发现",
```

## 四、比赛成果如何复用

| 比赛产出 | 转换为什么 | 用途 |
|---------|-----------|------|
| `01-域名与IP资产.md` | `fingerprint/` + `collectors/ip138.py` | 任意企业域名的IP反查规则 |
| `02-子域名完整清单.md` | `fusion/dedup.py` 去重算法 | 多源子域名去重合并 |
| `03-技术栈分析.md` | `fingerprint/*.py` 指纹库 | Coremail/CAS/Synjones自动识别 |
| `04-敏感信息与风险发现.md` | `reporting/report_generator.py` | P0-P2风险分级报告模板 |
| `05-人员与组织信息.md` | `social/` + 邮箱命名规则引擎 | 自动提取联系人 |
| `07-GitHub与代码泄露.md` | `collectors/github.py` 搜索规则 | GitHub泄露自动化监控 |
| `08-移动端与社交媒体资产.md` | `collectors/wechat.py` | 公众号/小程序自动发现 |
| `09-DNS深度信息与基础设施.md` | `collectors/` DNS源 | DNS/SPF/MX自动解析 |
| `10-深度技术探测与CMS指纹.md` | `fingerprint/` 指纹签名库 | 指纹匹配引擎 |
| `11-iFAFU非官方APP分析.md` | 第三方关联资产发现模块 | 非官方APP/OSS发现 |
| `12-凭证策略验证与工具说明.md` | 验证器规则库 | 弱密码策略检查 |
| `fafu_auto_verify.py` | `verifier/pipeline.py` 补充 | HTTP可达性验证 |
| `比赛报告_FAFU.json` | `reporting/templates/competition.json` | 比赛报告模板 |
